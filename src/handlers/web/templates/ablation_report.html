Глосарій термінів

Натисніть на термін, щоб побачити пояснення.

Сценарій (scenario)
Передзадана задача для генератора: тема + метр + кількість стоп + схема рим (наприклад, «весна, ямб 4-стопний, ABAB»). На сьогодні в системі 18 сценаріїв трьох категорій — Normal (типові), Edge (граничні: рідкісні метри / схеми / довгі вірші) і Corner (деґенеративні: «нуль стоп», тощо).
Конфіг (config) A–H
Варіант пайплайну з різним набором увімкнених стадій. A — baseline (тільки LLM + валідатор, без RAG, без feedback). E — повна система (RAG + metric examples + feedback) — рекомендований дефолт для типового використання. B / C / D — проміжні комбінації, щоб виміряти внесок кожного компонента окремо. F / G / H — ті ж збагачення, що C/D/E, але без feedback-циклу — для виміру чистого raw-впливу збагачень на перший драфт (research-only).
Seed
Випадкове початкове число для LLM (та інших стохастичних компонентів). Один прогон із сценарієм S, конфігом C і seed=1 дає один вірш; з seed=2 — інший. Кілька seeds потрібні, щоб усереднити випадковість моделі.
Прогін (run)
Один повний запуск пайплайну: scenario × config × seed. Стандартний батч — 18 × 8 × 3 = 432 прогони.
Paired delta (парна різниця)
Різниця метрик одного й того ж сценарію та seed-у між двома конфігами. Цей дизайн вимикає шум «складних сценаріїв» (наприклад, дактиль 6-ст. — об'єктивно важчий, неважливо для якого конфігу), і залишає лише ефект самої зміни конфігу.
95% bootstrap CI (довірчий інтервал)
Діапазон, у якому з імовірністю 95% знаходиться справжній середній ефект конфігу — оцінений ресемплінгом наявних спостережень. Якщо CI не перетинає 0 — ефект статистично значущий.
Wilcoxon p-value
Імовірність побачити такий або сильніший ефект випадково, якщо насправді конфіг нічого не змінив. p < 0.05 — традиційний поріг «значущо».
Forest plot
Графік із горизонтальними «вусами»: для кожного компонента — крапка (середній внесок) і відрізок (95% CI). Усе ліворуч від 0 — компонент шкодить, праворуч — допомагає.
Box plot
Розподіл метрики (наприклад, accuracy метру) по конфігах: коробка показує квартилі, лінія — медіану, «вуса» — діапазон.
Heatmap (config × scenario)
Таблиця: рядки — сценарії, стовпці — конфіги, клітинки кольоровані за результатом. Видно, який конфіг де працює добре, а де провалюється.

Як читати цей звіт

Звіт побудовано на paired-Δ дизайні: для кожного компонента ми беремо два конфіги, що відрізняються рівно цим компонентом (наприклад, B vs A — лише feedback-цикл), і дивимося різницю метрики на однакових (сценарій, seed) парах. Парування скасовує шум «цей сценарій просто складний» / «цього разу LLM повезло» — залишається саме внесок компонента.

Для кожного Δ рахуємо три речі:

  • Mean Δ — середнє покращення (в одиницях метрики, від 0 до 1).
  • 95% bootstrap CI — діапазон, у якому з 95% впевненістю лежить справжнє середнє ({{ artifacts.metadata.bootstrap_iterations }} ресемплінгів).
  • Wilcoxon p-value — статтест: чи відхилення від нуля систематичне.

«Значущо» = CI повністю по один бік від нуля. Якщо CI перетинає 0 — навіть при ненульовому середньому ефект непостійний, компонент допомагає одним сценаріям і шкодить іншим.
