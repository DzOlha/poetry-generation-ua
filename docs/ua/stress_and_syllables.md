# Наголос і склади

> Фундамент, на якому стоїть валідація метру. Без точного наголосу немає точного метру — усе інше «падає», бо метр визначається саме через позиції наголошених складів.

## Що потрібно від цієї підсистеми

- Сказати **кількість складів** у слові.
- Сказати **позицію наголошеного складу** у слові (0-based індекс).

Перше — тривіальне (рахуємо голосні). Друге — складне: потребує або словника, або евристики.

## Склади: як рахуються

Українською склад формується навколо голосного звука. Тому `count_syllables(word)` — це просто **кількість голосних літер** у слові.

**Реалізація:** [`shared/text_utils_ua.py`](../../src/shared/text_utils_ua.py) — функція `count_syllables_ua(word)` плюс канонічна константа `VOWELS_UA = "аеєиіїоуюя"`. Інфраструктурний адаптер [`UkrainianSyllableCounter`](../../src/infrastructure/stress/syllable_counter.py) обгортає цю функцію за портом `ISyllableCounter`, тож викликачі юніт-тестуються зі stub-counter-ами.

```
"весна"  → 2 (е, а)
"ліс"    → 1 (і)
"україна" → 4 (у, а, ї, а)
"сон"    → 1 (о)
```

Дифтонги типу «юа» рахуються як два склади (так як «ю» і «а» — окремі голосні). Це — свідоме спрощення: для української така класифікація дає точні результати для >99% випадків.

## Наголос: чому це складно

Український наголос **вільний і рухомий**:
- вільний = може падати на будь-який склад,
- рухомий = у різних формах того ж слова може бути різним (*вікнО → вікОн*).

Тому правила «наголос завжди на останньому/передостанньому» **не працюють** як єдине рішення. Правильна відповідь — **дивитися у словник**.

## Рівень 1: словникова перевірка

Проєкт використовує відкриту бібліотеку [`ukrainian-word-stress`](https://github.com/lang-uk/ukrainian-word-stress) (lang-uk) + Stanza NLP-моделі. Словник містить ~2 млн форм зі знятою двозначністю. Обʼєм — близько 1 ГБ розпакованих даних.

**Обгортка:** [`UkrainianStressDict`](../../src/infrastructure/stress/ukrainian.py) — реалізація порту `IStressDictionary`. Основний метод:

```python
def get_stress_index(word: str) -> int | None:
    """Return 0-based vowel index of stressed vowel, or None if unknown."""
```

Нюанси:
- Stressifier повертає слово з **комбінованим акутом над наголошеним голосним**. Обгортка перетворює це назад на індекс голосного (0-based).
- Повертається **індекс голосного**, не індекс складу. У слові "украї́на" (наголос на 3 голосну) метод поверне `2`.
- На **неоднозначних** словах (`зА́мок` vs `замО́к`) за замовчуванням використовується `on_ambiguity="first"` — беремо першу інтерпретацію. Стратегія конфігурована (`first` / `last` / `random`).
- Якщо бібліотека `ukrainian-word-stress` не вантажиться — конструктор пише warning-лог, а `get_stress_index` повертає `None`; верхній `IStressResolver` далі вмикає евристичний fallback.
- Інстанс Stressifier створюється на перший виклик і **кешується у module-level dict за ключем `on_ambiguity`** — щоб кілька `UkrainianStressDict` (наприклад, паралельні composition-контейнери у тестах) ділили один backend, не дублюючи важку модель.

## Рівень 2: евристика, коли словник мовчить

[`PenultimateFallbackStressResolver`](../../src/infrastructure/stress/penultimate_resolver.py) реалізує порт `IStressResolver`, обгортає словник і додає fallback для слів, яких у ньому немає (власні назви, рідкісні неологізми, друкарські помилки).

Алгоритм `resolve(word) -> int`:

1. **Кеш.** Якщо `word` уже був оброблений — повертаємо з `_cache`.
2. **Словник.** Викликаємо `stress_dictionary.get_stress_index(word)` (ін'єктований `IStressDictionary`). Якщо повернуло число — це воно.
3. **Fallback `_guess_stress(word)`:**
   - Один склад → повертаємо `0`.
   - **Правила суфіксів.** Якщо слово ≥3 складів і закінчується на `-ота` → наголос на **останній** склад. Список суфіксів (`_SUFFIXES_LAST_STRESS`) розширюваний.
   - **Правило кінцевої літери:**
     - Слово закінчується на голосну, `й`, або `ь` (**мʼяке закінчення**) → наголос на **передостанній** склад.
     - Слово закінчується на приголосну (**тверде закінчення**) → наголос на **останній** склад.

**Чому саме ця евристика.** Українська фонологія має статистичний ухил: слова з мʼяким закінченням у ~79% випадків мають наголос на передостанньому складі (Dolatian & Guekguezian, *Cambridge Phonology* 2019). Суфіксальні правила ловлять відому підмножину винятків. Разом це дає точність ~85–90% на невідомих словах — достатньо для fallback-режиму.

## Приклади

```
"сон"      → 0 (1 склад)
"весна"    → 1 (2 склади, тверде закінчення → останній)
"книжка"   → 0 (2 склади, мʼяке закінчення → передостанній)
"україна"  → 2 (4 склади; словник → "і")
"прапор"   → 0 (2 склади, тверде → останній? Ні: словник знає — "пра́пор")
"работа"   → 2 (3 склади; -ота → суфіксальне правило → останній)
```

## Порти (Dependency Inversion)

Шар наголосу / складів / фонетики розбито на чотири вузькі порти, тож викликачі залежать від контракту, а не від конкретного адаптера:

| Порт | Продакшн-адаптер | Відповідальність |
|------|------------------|------------------|
| `IStressDictionary` | `UkrainianStressDict` | Vowel-index lookup через lang-uk Stressifier |
| `ISyllableCounter` | `UkrainianSyllableCounter` | Підрахунок складів через голосні |
| `IStressResolver` | `PenultimateFallbackStressResolver` | Композиція двох вище + евристика + per-word кеш |
| `IPhoneticTranscriber` | `UkrainianIpaTranscriber` | Кирилиця → IPA + `vowel_positions` + `rhyme_part` |

`PhoneticRhymePairAnalyzer` залежить від `IStressResolver`, `ISyllableCounter`, `IPhoneticTranscriber` і `IStringSimilarity`; меньше-валідатори — опосередковано через `UkrainianProsodyAnalyzer`.

## Як це стикується з валідацією метру

Меньше-валідатор будує **реальний шаблон** рядка ось так:

1. Токенізація рядка на слова.
2. Для кожного слова:
   - Рахуємо кількість складів.
   - Резолвимо наголос (через `PenultimateFallbackStressResolver`).
   - У позицію наголошеного складу (у межах рядка) ставимо `—`, решта — `u`.
3. Виняток: **слова зі слабким наголосом** (сполучники, прийменники, частки — «і», «та», «на», «не»…) **не ставлять наголоси**. Вони проходять як нульові у метричній сітці.

Детальніше — див. [`meter_validation.md`](./meter_validation.md).

## Тонкощі та обмеження

- **Стабільний однопотоковий кеш:** `UkrainianStressDict` тримає інстанс Stressifier у модуль-рівневій змінній. У multi-worker режимі (uvicorn з workers >1) модель перезавантажується для кожного процесу. Це окей для прод (коефіцієнт старту), але не ідеально.
- **`on_ambiguity="first"`** — втрачаємо до ~2% випадків, коли перша інтерпретація неправильна. Компенсується толерантністю валідатора.
- **Відсутність словника для рідкісних форм.** Слово «загорілся» (рос. інтерференція, помилкова форма) — словник видасть `None` → евристика поставить наголос «як для звичайного слова». Результат може не збігтися з наміром автора.
- **Малі / великі літери.** Словник обробляє обидва регістри, кеш ключ — як є.
